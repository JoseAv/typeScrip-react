## Use State

para tipar un use state tenemos que indicarle el tipo de esta forma

``` ts
const [count,setCount]= useState<number | string>(0)
```

Ahora estos tipo de datos son primitivos pero que pasa cuando usamos un objeto una estructura unica, necesitamos decirle que forma tendra esta estructura, pero podriamos querer que primero sea null y luego tenga nuestras estructura

ejemplo

```js
type User = {name:string,age:number}
const [user, setUser] = useState<User | null>(null)  

// esto nos dara un error ya que nosotros le hemos dicho que puede ser null entonces no puede hacer null.age esto es error 
console.log(user.age)
console.log(user.name)


//Aqui colocamos este paramtro para que haga la comprobacion si user existe entonces si puede buscar a age
console.log(user?.age)
console.log(user?.name)

```

## Tipando setState

para tipar un setState no debemos de hacerlo como si fuera una funcion ya que nos dara un error, cada hook tiene su forma de ser tipado Por ejemplo


```ts
const [count,setCount]= useState<number | string>(0)

// ❌❌ esto estaria mal ya que setCount no es una funcion normal
const Prueba = ({setCount}: {setCount: ()=>void})=>{
    
    return (
        <button onClick= {()=> setCount(10)} ></button>
    )
}

// ✅✅ Debemos tipar bien el setCount

const Prueba = ({setCount}: {setCount: Dispatch<SetStateAction<number>>})=>{
    
    return (
        <button onClick= {()=> setCount(10)} ></button>
    )
}

```

Si bien se ve bastante complejo el IDE que uses te debe de indicar que tipo es setState para usarlo pero toma en cuenta que esas propiedades viene de react y tendras que importarlas.
