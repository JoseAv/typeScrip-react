# Operador As

As nos ayuda a indicarle a TS que nosotros sabemos que algun valor indicado es del esta interface o type, como alguna respuesta de un API donde tengo la certeza de lo que me regresa

``` ts
// unknown es para indicar que no sabemos que es, diferente a any que permite todo este no tendria operaciones
let status:unknown = 200

// como yo se que status es de tipo number podemos indicarle a ts que este tipo de number de esta forma
let newStatus = status as number

```

Y con Objeto?


``` ts
type user = {name:string, age:number}

// respuesta de una api
let api: unknown = {name:'Jose', age:20}

// Indicamos que tipo es la respuesta asi

let newUser = api as user
```

Hay que tener cuidado ya que aqui nos estamos saltando la verificacion de TS nosotros le estamos diciendo que es y no estamos tipando ya que podriamos tipar algo que no es, y el lo creeria como verdadero

``` ts 
type user = {name:string, age:number}

// respuesta de una api
let api: unknown = {status:200, data:{name:'Jose', age:20}}

// Indicamos que tipo es api

let newUser = api as user

// ahora esto es incorrecto porque cree que el objeto api es solo user

newUser.name // ❌ Error
newUser.age // ❌ Error

// Forma correcta

let newUser = api.data as user


newUser.name // ✅ ahora si funciona
newUser.age // ✅ ahora si funciona

```

# As con HTML

En js cuando nosotros queremos cambiar un elemento HTML realizamos un getElementById donde vamos a tomar un elemento html y sus propiedades pero con as podemos indicar que tipo es

``` js
// aqui tomamos el elemento y lo asignamos
const buttonHola = document.getElementById('hola') as HTMLButtonElement


<button id='hola'>Click</button>


```

# As Reconversion

Con as Tambien podemos reconverir un elemento que ya tiene un tipo y pasarlo a otro sin que nos de error, esto lo hacemos de esta forma


``` js
type user = {name:string}

const newUser:user = {name:'jose'}

// Reconvertimos

const numberUser = newUser as unknow as string[]

// Ahora este elemento puede hacer esto

numberUser.length

// Esto nos dara error pero es util si quieres pasar de un valor a otro

```
